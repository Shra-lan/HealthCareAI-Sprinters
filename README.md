## 📄 File Descriptions

### `smarttriage_v4.py` — The Algorithm
The complete V4 tournament algorithm. Runs **5 strategies in parallel**, compares their total risk scores, and returns the winner automatically.

| Strategy | Formula | Best For |
|---|---|---|
| `wspt` | `severity ÷ treatment_time` | Most datasets — short critical tasks first |
| `aging` | `severity × (1 + 0.2 × wait)` | Congested datasets — prevents starvation |
| `wspt_aging` | Hybrid of both | Mixed arrival patterns |
| `severity_only` | `severity` | Spread-out arrivals |
| `conservative_g` | WSPT + careful Doctor_G | Heavy TRAUMA-load datasets |

### `patients.csv` — Sample Input
The sample dataset provided in the problem statement. Use this to test the algorithm locally.

### `submission.json` — Sample Output
The generated output from running V4 on `patients.csv`. This is the file format judges expect.

## 📋 Input Format — `patients.csv`

Your CSV file must have exactly these 5 columns:

| Column | Type | Constraints |
|---|---|---|
| `patient_id` | string | Unique identifier (e.g. P1, P002) |
| `severity` | integer | 1 to 5 (5 = most critical) |
| `arrival_time` | integer | Minutes from start (≥ 0) |
| `treatment_time` | integer | Duration in minutes (≥ 1) |
| `required_specialization` | string | Must be `TRAUMA`, `CARDIO`, or `GENERAL` |

### Example `patients.csv`:
```csv
patient_id,severity,arrival_time,treatment_time,required_specialization
P1,5,0,8,TRAUMA
P2,3,2,4,CARDIO
P3,4,3,6,TRAUMA
P4,2,5,3,GENERAL
P5,5,7,5,TRAUMA
P6,1,8,2,GENERAL
```

### Output Rules:
- `treatments` sorted by `start_time` ascending
- `end_time` = `start_time` + `treatment_time` for every patient
- Every patient from the input appears **exactly once**
- `estimated_total_risk` = sum of `severity × (start_time − arrival_time)` for all patients

## 👨‍⚕️ Doctor Rules

| Doctor | Treats | Specialization |
|---|---|---|
| `Doctor_T` | TRAUMA patients only | Specialist |
| `Doctor_C` | CARDIO patients only | Specialist |
| `Doctor_G` | TRAUMA + CARDIO + GENERAL | Wildcard |

- Each doctor treats **one patient at a time** — no parallel treatment
- All doctors start **idle at time 0**
- Treatment **cannot be interrupted** once started

---

## ✅ The 4 Validation Rules

The algorithm validates all 4 rules automatically before producing output:

| Rule | Description | Disqualifies? |
|---|---|---|
| **Rule 1** — No Early Treatment | `start_time >= arrival_time` for every patient | ✅ Yes |
| **Rule 2** — No Doctor Overlap | One doctor treats one patient at a time | ✅ Yes |
| **Rule 3** — No Duplicates | Every patient appears exactly once | ✅ Yes |
| **Rule 4** — Valid Assignments | Doctor_T=TRAUMA only, Doctor_C=CARDIO only | ✅ Yes |

## 🧠 How the Algorithm Works

```
patients.csv
     ↓
Read + Parse patients
     ↓
Run 5 strategies in parallel:
  ├── Strategy 1: WSPT
  ├── Strategy 2: Aging
  ├── Strategy 3: WSPT + Aging
  ├── Strategy 4: Severity Only
  └── Strategy 5: Conservative Doctor_G
     ↓
Compare risk scores → Pick winner
     ↓
Validate all 4 rules
     ↓
submission.json  ✅
```

### Priority Score (how patients are ranked):
- **WSPT:** `severity ÷ treatment_time` — prefers short, critical tasks
- **Aging:** `severity × (1 + 0.2 × wait_time)` — score grows as patient waits longer

### Doctor_G Smart Overflow:
- **Aggressive mode:** Steals TRAUMA/CARDIO the moment their specialist is busy
- **Conservative mode:** Only steals if `specialist_wait > patient.treatment_time`
- Tournament automatically picks whichever mode scores lower
